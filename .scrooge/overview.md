**AA Grapher** is a static HTML+JS scatter-plot dashboard for comparing hundreds of AI models across benchmark scores, pricing, and latency.

Architecture: `clean.py` scrapes a Next.js RSC payload (`providers`), merges EQBench 3 chart data (`eqbench3_chartdata.js`) and LiveBench CSV (`table_2026_01_08.csv` with per-category aggregates from `categories_2026_01_08.json`) via normalized model-name matching with separate alias tables and stop-word sets for each source, aggregates host-level pricing/speed per model using the median, collects per-prompt-length TTFC/speed stats (`{field}_{plen}_prompt`), computes `intelligence_index_run_cost` from token counts and per-1M-token prices, and writes `data.js` (two exported constants: `FIELDS` — field objects with `id`, `name`, `pretty_name`, `direction`, `scale` — and `DATA` — model rows using short field IDs like `f0`). `index.html` loads `data.js` and Plotly 2.35.2 to render an interactive scatter plot with custom ARIA comboboxes, 2D/3D Pareto frontier filtering, creator/country/field-range filters, a leaderboard of models ranking #1 on multiple metrics, and URL-shareable state.

Non-obvious invariants: field `direction` defaults are heuristic (`price|cost|time|latency|hallucination|tokens|num_incorrect|num_not_attempted` → `min`; `context_window_tokens`, `non_hallucination` → `max`); `scale` defaults to `log` for price/cost/time/latency/tokens/num_hosts; `estimated_*` fields fall back to their base field then are deleted; `EXCLUDED_FIELDS` drops saturated/superseded benchmarks (aime, aime25, math_500, mmlu_pro, livecodebench, humaneval); `sig4` rounds floats to 4 significant figures; `keep_field` filters out `lab_claimed_*`, token count details, and partial sub-metric breakdowns (including non-total omniscience and briefcase sub-breakdowns); creator country emoji is derived from a hardcoded `CREATOR_FLAG` map; LiveBench model IDs use a separate normalizer (`_lb_norm`) with its own stop words (`_LB_DROP`) and alias table (`LB_ALIASES`); leaderboard Pareto badges mark cost/speed/size vs. Intelligence Index trade-offs; the entire app runs with no server and no build step.
